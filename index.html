about me

I strive to create high-quality and engaging projects where every detail—from adaptive layouts to micro-interactions—comes together to form a harmonious and well-considered result. Over a year and a half of actively working on diverse web page layouts, I’ve developed a clear approach: code should be clean, performant, and easy to understand, even if it means rejecting unnecessary frameworks (Yes, in many projects, they simply aren’t needed). My portfolio includes 20 completed works, among them 4 multi-page websites, each fully responsive (down to 320px—the smallest screen you might still encounter), optimized for fast loading (with proper use of images via srcset, lazy loading), and enriched with thoughtful, dynamic effects (smooth hovers, scroll animations, custom transitions, and more). My experience with Tailwind CSS, React, and even Django (during a team project at HATACONE) has shown me that "heavy" solutions can often be replaced with lightweight ones—pure HTML, CSS, and JavaScript. This not only reduces reliance on external libraries but also makes the code more transparent for other developers, simplifying maintenance and scalability. Visually, there’s no difference: the user gets the same visual and functional interface, but "under the hood," it works faster and more reliably. My philosophy: if something can be done with native technologies without sacrificing quality—it’s an option worth considering. My particular strength lies in attention to nuances that often go unnoticed: managing content overflow on small viewports, ensuring accessibility of interactive elements (even in the :focus-visible state), selecting optimal image formats (WebP with fallbacks), and typography refinements (kerning, hyphenation). The end result isn’t just a "page from a mockup"—it’s a meticulously crafted, pixel-perfect product that performs equally well on an old smartphone and a 4K monitor.
